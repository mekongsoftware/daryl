Complete Restaurant Solution

Developed by Daryl Khuong

Technical Details

  • AngularJS 1.5
  • Bootstrap
  • Web API, Entity Framework
  • SQL Server
  • Microsoft Azure
  • Token based authentication, OWIN and Identity

Development Environment

  • Visual Studio 2015, any edition
  • SQL Server, any edition
  • SSMS, v 2016 is recomended
  • Git/BitBucket

Development Setup - Using Azure Backend

  • Modify config.js in CRS.Angular project change this line
    window.__env.baseUrl = 'http://guavasystem.azurewebsites.net';
  • Right click on index.html, select View in Browser to start the program

Development Setup - Local

  • Create the database locally. In SSMS, copy Create_RestaurantDb.sql from the solution, modify where the database file is located,
    FILENAME = N'D:\_sql_data\Restaurant.mdf' and
    FILENAME = N'D:\_sql_data\Restaurant_log.ldf'
    then Execute
  • Create roles, lookup table and customer test data. In SSMS, execute these 3 stored procedures:
    EXEC DeleteAllData
    EXEC RefreshData
    EXEC SetupRestaurantDataForTesting
  • Create user login using Identity. Run Web.API project, you'll see an error, it's expected. Change the URL to
    http://localhost:xxxx/api/CreateAccounts then enter. This will create initial accounts for each restaurant (demo, waiter, preparer, deliverer, and owner).
  • Modify config.js in CRS.Angular project change this line
    window.__env.baseUrl = 'http://localhost:xxxx'; where xxxx is the actual port of Web.API.
  • Right click on index.html, select View in Browser to start the program
